# AGRA / NOYA Website Modernization Project (Hand-off Document)

이 문서는 다른 담당자가 Antigravity(AI 코딩 어시스턴트)를 사용하여 **AGRA 일반 광고 웹사이트(HR 통합 시스템 제외)**의 지속적인 업그레이드와 유지보수를 원활하게 이어갈 수 있도록 작성된 인수인계 가이드입니다.

이 문서를 Antigravity 프롬프트 창에 복사해서 붙여넣거나, 프로젝트의 루트 폴더에 `HANDOFF_GUIDE.md` 등의 이름으로 저장해두고 Antigravity에게 "이 파일을 읽고 프로젝트 컨텍스트를 파악해 줘"라고 요청하시면 됩니다.

---

## 🏗️ 1. 기술 스택 (Tech Stack)

*   **프레임워크:** Next.js (App Router `src/app` 방식 사용)
*   **언어:** TypeScript (`.tsx`, `.ts`)
*   **스타일링:** 순수 CSS (`.css`, CSS Modules) 
    * *TailwindCSS는 명시적인 요청이 없는 한 사용하지 않는 규칙으로 개발됨.*
*   **데이터베이스 연결 (사내 시스템 한정):** Prisma ORM + SQLite (`prisma/schema.prisma`)
*   **주요 패키지:**
    *   `lucide-react`: 사이트 전반의 아이콘 부드럽게 구현
    *   `xlsx`: 엑셀 데이터 파일 파싱 (관리자 페이지용)
    *   `crypto-js`: 스토리지 데이터 암호화용

## 🎨 2. 디자인 방향성 (Design Aesthetics)

*   **컨셉:** Premium & Modern Indian (프리미엄, 모던 인디안 다이닝)
*   **다크 모드 베이스:** AGRA 오프라인 매장의 어두운 조명과 고급스러운 분위기를 반영하여, 순백색을 지양하고 깊은 다크 (#0a0a0a) 톤에 골드 (#D4AF37) 포인트를 사용합니다.
*   **타이포그래피:** 
    *   강조되는 헤딩 (Heading): 우아한 느낌의 Serif 폰트 (예: `Playfair Display`, `Nanum Myeongjo`)
    *   일반 본문 (Body): 가독성 좋은 Sans-Serif 폰트 (예: `Inter`, `Pretendard`)
    *   *관리자 페이지(`/admin/settings`)에서 디자인 설정(색상, 폰트)을 동적으로 변경할 수 있도록 `SettingsContext`를 사용 중입니다.*

## 📂 3. 프로젝트 주요 구조 (Project Structure)

프로젝트 폴더 구조 중 **일반 홍보 웹사이트**에 관련된 핵심 파일/폴더들입니다.
새로운 담당자는 이 파일들을 위주로 수정 및 업그레이드를 진행하면 됩니다.

```text
src/
├── app/
│   ├── layout.tsx         # 일반 웹사이트(고객용)의 글로벌 레이아웃 (네비게이션 바, 헤더/푸터 등)
│   ├── page.tsx           # 웹사이트 메인(Home) 페이지 (이미지 슬라이더 및 패럴랙스 효과 적용)
│   ├── menu/              # 메뉴 소개 페이지 (카테고리 필터링 및 상세 뷰어 모달 포함)
│   ├── location/          # 매장 안내 및 예약(Naver Booking) 링크 연결 페이지
│   ├── news/              # 새로운 소식 및 이벤트 배너 페이지
│   │
│   ├── admin/             # [마케팅/운영자용] 웹사이트 콘텐츠 관리자 페이지
│   │   ├── settings/      # 브랜드 테마(색상, 폰트), 로고, 히어로 이미지 변경 
│   │   ├── menu/          # 메뉴 사진, 이름, 가격 추가/수정/삭제 
│   │   ├── stores/        # 매장(지점) 주소, 영업시간, 예약 링크 관리
│   │   └── AuthContext.tsx # 관리자 로그인 세션 (비밀번호: Agra1234!)
│   │
│   ├── SettingsContext.tsx # (/admin)에서 변경된 설정값을 웹사이트 전체에 동적으로 전파하는 상태 전역 관리소
│   └── globals.css        # CSS 변수 및 전역 리셋 스타일
```

*(참고: `src/app/hr-portal-admin/` 및 `src/app/api/hr/` 폴더는 인사팀 직원들을 위한 별도의 사내 그룹웨어 시스템 코드로 개발 방향 및 권한이 마케팅 웹사이트와는 무관하게 분리되어 있습니다.)*

## 💡 4. 현재까지 구현된 핵심 기능 요약

1.  **동적 디자인 시스템 적용 완료**: 관리자 페이지(`/admin/settings`)의 변경 내역이 `SettingsContext`를 통해 실시간으로 메인 웹사이트의 CSS 변수(`var(--primary-color)`) 등을 업데이트합니다.
2.  **이미지 자동 슬라이더 (Homepage)**: `src/app/page.tsx`에 5초 주기로 페이드인/아웃되는 브랜딩 이미지 크로스페이드(Carousel) 기능과 패럴랙스 스크롤 효과가 구현되어 있습니다.
3.  **반응형 메뉴 갤러리 (`/menu`)**: 메뉴 클릭 시 화면 중앙에 크게 뜨는 팝업(Detailed View Popup)이 구현되어 있습니다. 이미지 최적화를 위해 브라우저 단에서 고용량의 음식 사진을 자동으로 `.webp`로 압축 변환하게 되어 있습니다.
4.  **매장 정보 통합 (`/location`)**: 지점별 정보와 네이버 플레이스 예약 버튼을 연동해둔 상태입니다.

## 🚀 5. Antigravity에게 효율적으로 업무를 지시하는 프롬프트 팁

새로운 담당자가 Antigravity에게 웹사이트 수정을 요청할 때, **아래 예시처럼 구체적으로 지시하면 개발 오류 확률이 획기적으로 낮아집니다.**

*   **[GNB / 네비게이션바 메뉴 수정 시]**:
    > "웹사이트의 상단 헤더와 하단 푸터에 새로운 링크(예: 브랜드 스토리)를 추가하려고 해. 기존 고객용 디자인 톤에 맞추어서, 관련 컴포넌트인 `src/app/layout.tsx` 파일과 글로벌 CSS를 먼저 확인하고 작업해 줘."
*   **[메인 페이지에 새로운 섹션 추가 시]**: 
    > "메인 화면(`src/app/page.tsx`) 하단에 최근 인스타그램 피드를 보여주는 섹션을 추가하려고 해. 디자인은 현재 사이트 분위기에 맞춰 어두운 배경과 골든 포인트 색상을 유지해야 하고, 모바일 화면(스마트폰 해상도)에서도 깨지지 않게 반응형으로 짜 줘."
*   **[마케팅 어드민(관리자) 기능 추가 시]**:
    > "소비자 화면에 프로모션 임시 팝업창 공지를 띄우려고 하는데, 일반 사용자들이 보는 팝업 UI를 먼저 만들고, 그 이미지와 내용을 `localhost:3000/admin` 쪽 관리자가 직접 등록하고 ON/OFF 할 수 있는 스위치를 추가해 줘. 관련 데이터는 로컬 스토리지 또는 컨텍스트에 전파되야해."
*   **[명확한 범위 및 제약조건 명시]**: 
    > "(중요) 이번 작업에서는 마케팅을 위한 오픈 웹사이트 디자인 기능만 수정해. `hr-portal-admin` 이나 직원들이 쓰는 내부 인사 파일 코드는 절대 건드리지 마."
